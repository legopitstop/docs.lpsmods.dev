# Submitting a good bug report

## 1. Describe the problem

What did you expect to happen?

## 2. Steps to reproduce

List the exact steps in order. If it only happens sometimes, say so.

## 3. One bug per report

Separate reports are easier to track and fix.

## 4. Be specific

Avoid vague reports. Bad examples:

- "It's broken"
- "Doesn't work"

## Template

```md
**Describe the bug:**

A clear and concise description of what the bug is.

**Steps to Reproduce:**

Steps to reproduce the behavior.

1.
2.
3.

**Expected Behavior:**

A clear description of what you expected to happen.

**Software Version:**

The version of the software you are using.

**Platform:**

The device or system you are using the software on.

**Additional Context:**

Add any other information about the problem. Include screenshots, videos, and logs if possible.
```
